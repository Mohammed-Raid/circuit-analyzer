# Détection d'îlots fonctionnels — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Regrouper les composants en îlots fonctionnels (connexité hors rails) et les exposer dans le rapport, l'export XML et la GUI.

**Architecture:** Nouveau module `circuit_analyzer/ilots.py` (Union-Find sur les refs, rails exclus), calcul unique en fin d'`analyser()`, résultat attaché à `ResultatsAnalyse.ilots`. Rendu : section rapport, ordre des blocs XML, panneau repliable GUI.

**Tech Stack:** Python 3.10+, NetworkX, pytest, CustomTkinter (GUI).

**Spec:** `docs/superpowers/specs/2026-06-10-ilots-design.md`

**Règles transverses (héritées du sous-projet satellites) :**
- Sortie rapport limitée à cp1252 (pas de `→ ⚠ ≥ ─`).
- Tests : `Component` / `build_graph` / `match_patterns` (mêmes imports que test_satellites.py).
- Jamais de Co-Authored-By Claude dans les commits.
